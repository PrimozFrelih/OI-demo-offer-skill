# Cenovna pravila

Cene v ponudbah morajo biti pregledne, razdeljene po smiselnih postavkah in usklajene z obsegom dela. Ne skrivaj večjih aktivnosti v eni sami skupni ceni, razen če uporabnik to izrecno zahteva.

## Osnovna pravila

- Vse cene piši v EUR.
- Uporabi slovenski zapis zneskov, na primer `3.600,00 EUR`.
- Vedno dodaj vrstico `Skupaj`.
- Vedno dodaj stavek `Cene ne vključujejo DDV.`
- Mesečne naročnine, vzdrževanje, gostovanje in licence navedi ločeno od enkratne projektne cene.
- Če je cena ocena, uporabi izraz `predvidena vrednost projekta`.

## Vzorec za ERP uvedbo

Referenčni vzorec za uvedbo ERP sistema:

| Postavka | Tipičen delež |
| --- | ---: |
| Analiza procesov in projektno vodenje | 15-25 % |
| Nastavitev okolja ali modulov | 25-40 % |
| Migracija podatkov | 10-20 % |
| Testiranje in podpora | 10-20 % |
| Usposabljanje | 5-15 % |
| Produkcijski zagon | 5-15 % |

Pri ERP ponudbah je lahko ločeno navedena mesečna naročnina za uporabo sistema, ki vključuje osnovno vzdrževanje, gostovanje, varnostne kopije in redne posodobitve.

## Vzorec za razvoj aplikacije

Referenčni vzorec za razvoj aplikacije:

| Postavka | Tipičen delež |
| --- | ---: |
| Analiza zahtev in projektno vodenje | 10-20 % |
| UX zasnova | 5-15 % |
| Razvoj uporabniškega dela | 30-45 % |
| Razvoj zaledja in API | 20-35 % |
| Testiranje in odprava napak | 8-15 % |
| Objava, uvajanje ali predaja | 5-10 % |

Vzdrževanje po zaključku projekta se praviloma navede kot mesečni paket in se lahko dogovori posebej.

## Plačilni mejniki

Plačilni mejniki naj sledijo izvedbenemu tveganju:

- avans ob potrditvi ponudbe: običajno 30 %
- vmesni mejnik po zasnovi, nastavitvi ali testni verziji: 20-40 %
- zaključni mejnik po produkcijskem zagonu ali zaključku projekta: 20-30 %

Če je projekt daljši ali dražji, uporabi več vmesnih mejnikov.

## Dodatna dela

Dodatna dela se ocenijo posebej, kadar:

- funkcionalnost ni navedena v ponudbi
- naročnik spremeni potrjeno specifikacijo
- se doda integracija z zunanjim sistemom
- so vhodni podatki nepopolni ali neustrezni za dogovorjeno migracijo
- se zahteva napredna analitika, večjezičnost, offline način ali posebna infrastruktura

Priporočena formulacija:

`Dodatne zahteve se obravnavajo kot sprememba obsega in se ocenijo posebej.`
